# Задание 5: Fuzzy kNN для скрининга преддиабета

## Клиническая задача
Разработать алгоритм для скрининга преддиабета в поликлинике с переходом от жёсткой классификации к персонализированной оценке риска.

## Требования к выполнению

### Часть 1: Подготовка целевой группы (15%)
- [ ] Отфильтровать пациентов с глюкозой 5.4-6.5 ммоль/л
- [ ] Описать распределение статусов глюкозы в группе
- [ ] Выбрать 5 признаков для модели

### Часть 2: Реализация Fuzzy kNN (35%)
- [ ] Реализовать класс `FuzzyKNN` с методами:
  - `fit(X, y)` — обучение
  - `predict_membership(X)` — степени принадлежности
  - `predict_risk_score(X)` — скор риска
- [ ] Параметры: k=7, m=2.0
- [ ] Разделить данные на train/test

### Часть 3: Анализ результатов (30%)
- [ ] Построить гистограмму скоров риска
- [ ] Визуализировать степени принадлежности для примера пациента
- [ ] Сравнить жёсткую и нечёткую классификацию
- [ ] Проанализировать неопределённость прогнозов

### Часть 4: Клинический протокол (20%)
- [ ] Разработать тактику наблюдения по порогам риска:
  - <30%: стандартная диспансеризация
  - 30-70%: усиленный контроль
  - ≥70%: направление к эндокринологу
- [ ] Проанализировать «серую зону» (глюкоза 5.8-6.0)
- [ ] Описать этические аспекты применения

## Критерии оценки

| Критерий | Баллы | Описание |
|----------|-------|----------|
| Класс FuzzyKNN реализован | 30 | Все методы работают |
| Визуализации (4 графика) | 25 | Полная информация |
| Анализ неопределённости | 20 | Выявлены пациенты в «серой зоне» |
| Клинический протокол | 25 | Готов к внедрению |

## Подсказки
- Скор риска = membership[преддиабет]*0.5 + membership[диабет]*1.0
- Неопределённость = 1 - max(memberships)
- Для «серой зоны» сравните бинарный и нечёткий подходы
